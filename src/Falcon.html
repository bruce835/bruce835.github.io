Brief Overview

Falcon syntax will draw heavily from C++, but with an aim to be as simple, consistent,
and coder-friendly as possible. It will be built with an aim on functional programming,
however I also want to support my own take on object-oriented programming: "Winging".

What is a wing? A wing is basically a box of information. When you attach an object
to a wing, you are not forcing that object into the box; rather, the object rides on
the wing, able to gain access to its information without being forced to follow
strict rules. I have yet to fully flesh out how these will work, and I haven't even begun
to implement them, but they have been my plan from the very start.

Design Principles

Below are the core principles of Falcon and key frustrations
I plan to address with it.

\
  1. 1. Rigid hierarchies should not be enforced.

    2. Objects and subclasses shouldn't be made to strictly fit their parents.

  2. 2. Let verbs be verbs.

    3. No weird objects like fstream. Objects should be used to represent complex data
    structures or real-world objects, not simple instruction sets.

  3. 3. Modularity

    4. Objects should be able to easily pick and choose what they want from
    a class without being made a child object or having to create a new local object of that class.

I will likely expand on these at a later date, but these are the main ideas.

and libraries. However, one can define a function "main" with a return type of int and

The Boring Stuff

Now, how do we use the Falcon compiler? (Note that the
falcon compiler is a linux exectuable, and you currently
have to compile the compiler yourself if you are using macOS. The compiler will not work on windows, as
it currently only knows the syscalls for linux and macOS.) First run the falcon
exectuable.

This generates the assembly(and prints a beautiful parse tree), but not the .o file. The assembly generated
is nasm x86. An easy way to run the falcon "compiler" and necessary nasm and ld
commands all at once is to execute one of the provided bash scripts (run_liunux.sh
and run_macos.sh).
